# 未注册光猫查询 - APK 打包指南

---

## ⭐ 方案 A：GitHub 一键构建（推荐，10 分钟）

### 第一步：上传代码到 GitHub

1. 打开 https://github.com/new
2. Repository name 填 `onu-query`，选 Private，点 **Create repository**
3. 页面不要关，稍后要改 Settings

### 第二步：上传文件

把 `workspace-android` 文件夹里的所有文件（main.py、buildozer.spec、icon.png 等）上传到这个仓库

### 第三步：开启 Actions 权限

- 仓库页面 → **Settings** → 左侧 **Actions** → **General**
- 滚动到页面底部，找到 **Workflow permissions**
- 选 **Read and write permissions** → 点 **Save**

### 第四步：触发构建

- 仓库页面 → **Actions** → 左侧 **🤖 Build Android APK**
- 右侧绿色按钮 **Run workflow** → 点 **Run workflow**
- 等待约 **10~15 分钟**（首次构建需下载 SDK）

### 第五步：下载 APK

构建完成后：
- 点击构建任务（绿色勾 ✅）
- 页面下方 **Artifacts** → 点击 **onu-query-apk** 自动下载

### 第六步：安装到手机

1. 把下载的 APK 传到手机
2. 手机：设置 → 安全 → 开启 **"允许安装未知来源应用"**
3. 点 APK 安装

---

## 📱 方案 B：手机 Termux 编译（无需电脑）

如果你有安卓手机：

1. **应用商店搜索 Termux** 安装
2. 打开 Termux，依次运行：

```bash
# 更新并安装 Python 和 Git
pkg update
pkg install python git

# 安装编译工具
pkg install openjdk-17

# 安装 Python 依赖
pip install buildozer==1.5.0 kivy==2.3.0 requests pillow

# 克隆代码（把代码上传到 GitHub 后复制仓库地址）
git clone https://github.com/你的用户名/onu-query.git
cd onu-query

# 编译 APK（首次约 20~30 分钟）
buildozer android debug
```

3. APK 生成在 `bin/` 目录下，用文件管理器找到安装

---

## ⚙️ 方案 C：Windows + WSL（本地编译）

如果想在本机编译，需要先安装 WSL：

1. **管理员权限打开 PowerShell**，运行：
```powershell
wsl --install -d Ubuntu
```
2. 重启电脑
3. Ubuntu 打开后运行构建脚本里的命令

## 常见问题

**Q: 构建失败怎么办？**
→ Actions 页面点击失败的 workflow，查看日志，常见错误是权限没开（重复方案 A 第三步）

**Q: APK 装不上？**
→ 确保手机开启了"允许安装未知来源应用"（设置 → 安全）

**Q: 查不到数据？**
→ 确认手机网络能访问 http://183.215.60.137:9082
